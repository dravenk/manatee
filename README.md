# Manatee - 安全数字货币支付系统

Manatee 是一个基于 TypeScript 的前后端分离的数字货币支付系统，集成了安全的钱包管理功能，为支付提供企业级安全保障。

## 🎯 核心定位

**Manatee 是一个集成了钱包管理的数字货币支付系统**

- **主要功能**：数字货币订单创建、支付和管理
- **基础能力**：集成钱包管理，确保支付安全
- **安全原则**：私钥和助记词永远不会以明文形式存储或传输

## 🔒 钱包安全（支付的基础保障）

### 核心安全原则

**您的私钥和助记词永远不会以明文形式存储或传输。**

### 软件钱包安全（企业级加密）

对于助记词和私钥钱包，系统采用**多层加密保护**：

- **Argon2id + AES-256**：双重加密算法，密钥仅在内存中使用
- **不持久化密钥**：解密后的私钥**从不**写入磁盘或数据库
- **密码验证**：每次使用都需要输入密码进行解密
- **加密存储**：所有私钥和助记词都经过加密后存储在服务器 `private/` 目录

## 🚀 功能特性

### 订单支付系统

- **订单创建**：创建数字货币支付订单
- **钱包支付**：使用连接的钱包安全支付订单
- **订单管理**：查看订单状态和历史记录
- **多币种支持**：支持 USDT (TRC20)、TRX 等

### 钱包管理（支付基础）

- **多钱包支持**：同时管理多个硬件钱包和软件钱包
- **多账户派生**：从单个钱包派生多个账户（BIP44 标准）
- **多网络支持**：支持 TRON、Ethereum、Bitcoin 等多个区块链网络

## 📦 技术栈

- **前端**：React 19 + TypeScript + Tailwind CSS
- **后端**：Bun + TypeScript
- **加密**：Argon2id + AES-256
- **钱包标准**：BIP32、BIP39、BIP44 等
- **部署**：Docker Compose

## 🛠️ 快速开始

### 初始化配置

```bash
# 初始化环境变量配置（设置端口等）
make setup
```

这会从 `.env.example` 创建 `.env` 文件，您可以编辑这些文件来配置：
- `server/.env` - 后端服务配置（本地和 Docker 共用）
  - `PORT` - 本地开发使用的端口
  - `SERVER_PORT` - Docker 部署使用的端口（用于端口映射）
- `frontend/.env` - 前端服务配置（本地和 Docker 共用）
  - `PORT` - 本地开发使用的端口
  - `FRONTEND_PORT` - Docker 部署使用的端口（用于端口映射）
  - `VITE_API_BASE_URL` - 后端 API 地址（Docker 环境使用服务名）

### 本地开发环境

```bash
# 安装依赖
make install

# 启动本地开发环境（前端和后端）
make start
```

服务地址从 `server/.env` 和 `frontend/.env` 中的 `PORT` 读取（默认 6543 和 3456）。

### Docker 生产部署

```bash
# 一键部署（自动运行 setup 并从 .env 读取端口配置）
make deploy
```

Docker 部署会从 `server/.env` 和 `frontend/.env` 中读取 `SERVER_PORT` 和 `FRONTEND_PORT` 进行端口映射。

其他命令：
- `make stop` - 停止所有服务
- `make logs` - 查看服务日志
- `make clean` - 清理构建文件和依赖
- `make help` - 查看所有可用命令


## 📖 使用指南

### 1. 连接钱包（支付的前提）

#### 硬件钱包（推荐）
1. 在钱包管理页面选择"连接硬件钱包"
2. 选择钱包类型并连接
3. 系统只保存地址和公钥，**不保存私钥**

#### 软件钱包
1. 导入助记词或私钥
2. 设置强密码用于加密
3. 系统使用 Argon2id + AES-256 加密存储

### 2. 创建和支付订单

1. **创建订单**：在订单页面填写金额、币种和描述
2. **连接钱包**：确保已连接硬件钱包或导入软件钱包
3. **支付订单**：点击"支付订单"，在钱包中确认签名
4. **完成支付**：签名完成后订单状态更新为"已支付"

## 🔐 安全保证

### 支付安全

- ✅ **钱包连接验证**：所有支付操作都需要钱包连接
- ✅ **签名在钱包中完成**：硬件钱包在设备上签名，软件钱包在内存中签名后立即清除
- ✅ **系统无法代替签名**：系统无法代替您签名交易

### 钱包安全

- ✅ **硬件钱包**：私钥永远不离开硬件设备
- ✅ **软件钱包**：私钥经过 Argon2id + AES-256 双重加密，密钥只在内存中使用
- ✅ **访问控制**：所有钱包操作都需要用户身份验证

## 📝 常见问题

### Q: 我的私钥安全吗？

**A: 非常安全！**

- **硬件钱包**：私钥永远不离开硬件设备
- **软件钱包**：私钥经过双重加密，密钥只在内存中使用，从不持久化

### Q: 如果服务器被攻击，我的资产会丢失吗？

**A: 不会！**

- **硬件钱包**：私钥不在服务器上
- **软件钱包**：即使攻击者获取加密数据，没有您的密码也无法解密

### Q: 订单支付安全吗？

**A: 是的**

- 所有交易签名都在您的钱包中完成
- 系统无法代替您签名交易
- 支付前会显示交易详情供您确认
